業務自動化システム導入のご提案

1. 要件

対象: Amazon（セラーセントラル）、中国ECサイト、ネクストエンジン間の連携とファイル操作。

環境・運用: Mac/スマホから簡単に操作・確認ができること。指定時刻の自動実行。

エラー対応: データ不備などのエラーを自動検知し、適切に通知・リカバリを行うこと。

コスト: ランニングコスト（維持費）を極限まで抑えること。

2. ご提案プラン：Python ＋ GitHub Actions (Issue Ops)

画面操作による不安定なRPAツールではなく、**各システムの「API」を直接連携させる専用プログラム（Python）**を開発します。

安定稼働: 画面デザインの変更に影響されず、API通信で確実にデータを処理します。

直感的な運用 (Issue Ops): 掲示板（Issue）にコメントするだけで、実行指示や結果の受け取り、履歴管理が可能です。

維持費実質0円: クラウド（GitHub Actions）の無料枠を利用し、サーバー代やツール利用料をゼロにします。

3. エラー発生時の運用イメージ（例：未入力エラー）

[実行] 担当者がIssueに「実行」とコメント。システムがクラウド上で自動起動。

[検知] API通信で「〒が未入力」というエラーを自動検知。

[通知・停止] システムがIssueに「〇〇様の〒が未入力です。修正して再実行してください」と返信して待機。
